# 5.20 - 6.01 冲刺阶段划分

> [!summary]
> 目标：围绕华为风格二面，完成项目深挖、技术防御、算法肌肉记忆与高压模拟四段式冲刺。

## 阶段总览

### 第一阶段（Day1-Day3）：打开黑盒
- 核心目标：梳理三大核心项目的全链路，消灭简历上的所有认知盲区。
- 输出成果：Obsidian 中的 3 张核心业务时序图 + 3 份口述录音。

### 第二阶段（Day4-Day6）：防御构建
- 核心目标：将项目中用到的中间件和 AI 框架与底层原理绑定，准备高频追问库。
- 输出成果：Obsidian 中的场景化八股文问答对。

### 第三阶段（Day7-Day9）：肌肉记忆
- 核心目标：建立题型识别能力，掌握核心数据结构的套路模板。
- 输出成果：Obsidian 手撕题模板库 + 每日 2 道核心算法讲解录音。

### 第四阶段（Day10-Day13）：高压对练
- 核心目标：适应华为风格的深挖节奏，锻炼临场反应和话术组织。
- 输出成果：稳定的面试状态，录音复盘后的错题集。

## 每日详细计划

### Day 1（5.20）：死磕 MCP 代码智能体引擎
- 每日优先级：P0，必须彻底打通该项目。
- 每日耗时预估：6 小时。
- 上午：CodeX 辅助阅卷。把 MCP 引擎源码喂给 CodeX，只问一个问题：“梳理一个复杂代码修改指令从输入到执行完毕的完整时序图”。重点分析 ReAct 状态机流转和 GitWorktree 挂载逻辑。
- 下午：Obsidian 绘图与话术构建。在 Obsidian 中用 Mermaid 画出上午梳理的时序图。写下“项目背景”“我的职责”“核心链路”“为什么用 GitWorktree 而不是 Docker”的逐字稿。
- 晚上：闭卷口述训练。不看任何资料，用手机录音，给自己讲 10 分钟这个项目的全流程。听录音，找出卡壳的地方，在 `04_高频追问与防坑` 记录。
- 每日验收标准：能不看图，顺畅画出系统的架构白板图，并口述数据流。
- 如何避免假学习：不要抄网上的 GitWorktree 概念，必须结合你自己的目录挂载和隔离逻辑来讲。

### Day 2（5.21）：打通 AI Coding 独立全栈高并发链路
- 每日优先级：P0。
- 每日耗时预估：6 小时。
- 上午：CodeX 链路还原。重点分析“高并发秒杀与解耦”模块。用 CodeX 跑通 `Controller -> Redis（预扣库存）-> Lua -> RocketMQ -> 数据库` 的调用链。
- 下午：技术防坑准备。梳理 MQ 消息积压怎么办、Redis Lua 脚本执行超时怎么办、Caffeine + Redis 双写一致性怎么保证，写入 Obsidian 库。
- 晚上：口述与录音复盘。录音 10 分钟，向“面试官”解释你是如何用 RocketMQ 削峰填谷，并大幅降低接口 RT 的。必须有具体数据支撑，哪怕是压测估算数据。
- 每日验收标准：能清楚解释 Lua 脚本在项目里具体校验了什么字段，MQ 的消费端幂等是怎么做的（唯一键 / 防重表）。

### Day 3（5.22）：梳理中科院 / 石油研究院项目链路
- 每日优先级：P1。
- 每日耗时预估：5 小时。
- 上午：复盘图谱混合检索链路。Neo4j + ES 混合召回的数据同步怎么做的，Kafka 异步流处理的 Topic 是怎么设计的。
- 下午：梳理非结构化 LAS 文件解析与 MyBatis-Plus 批量入库的锁竞争问题排查过程。
- 晚上：将这三个项目串联，提炼出一个共通的“自我介绍”话术。突出你“既懂 AI 应用落地，又具备扎实后端工程能力”的人设。
- 每日验收标准：能够用 STAR 法则精准描述在软件所解决的“长任务 HTTP 超时”问题。

### Day 4（5.23）：技术栈深挖 - Redis 与 RocketMQ
- 上午：结合项目背八股。不要干背理论。思考：AI Coding 里的 Redis 挂了怎么兜底，RocketMQ 如果发送半消息失败怎么回滚。
- 下午：在 Obsidian 整理这两者的“高频追问库”及对应话术。
- 晚上：与教练进行 30 分钟专项对练，只针对缓存和 MQ 疯狂追问项目边界。

### Day 5（5.24）：技术栈深挖 - 大模型应用框架与图数据库
- 上午：复习 LLM 基石理论（结合笔试题中的 BPE 算法）和 ReAct 范式底层。梳理 Neo4j 的基础查询语法和图遍历优化。
- 下午：梳理“Token 截断机制”“防御模型发散”的具体实现代码。
- 晚上：专项对练。重点追问：如果大模型死循环调用同一个 Tool，状态机是怎么拦截的。

### Day 6（5.25）：框架底层与系统调优
- 上午：Spring Boot 启动流程、Spring Security + JWT 鉴权底层逻辑。
- 下午：MySQL 索引优化（结合批量入库和复杂表查询）、JVM 基础排错。
- 晚上：专项对练。从 SSO 鉴权切入，深挖到分布式 Session 和底层拦截器链路。

### Day 7（5.26）：手撕专项 - 数组与滑动窗口
- 上午：刷“合并有序数组”“滑动窗口最大值”。
- 下午：不只是写，练习边写边讲。在 Obsidian 记录模板和错点，如边界越界。
- 晚上：口述滑动窗口最大值的双端队列解法逻辑。

### Day 8（5.27）：手撕专项 - 字符串、Trie 与位运算
- 上午：重写并讲透 BPE 分词算法模拟题、实现 Trie、只出现一次的数字 II。
- 下午：整理树与图的基础遍历（DFS / BFS）模板，防备图谱项目被引申。
- 晚上：口述 Trie 树的插入和查询时间复杂度，以及 BPE 贪心合并的逻辑。

### Day 9（5.28）：手撕专项 - 链表与栈 / 队列
- 上午：旋转链表、基本计算器 II。
- 下午：复习所有算法模板，整理一份考前 10 分钟必看速记表。
- 晚上：综合代码考核。随机抽两题，在规定时间内口述思路并给出核心代码结构。

### Day 10（5.29）：系统设计与场景扩展
- 全天重点：应对开放性问题。
- 重点题：如果 AI Coding 系统日活突然增长 10 倍，当前架构哪里最先崩溃，如何改造。
- 输出框架：从网关限流、缓存扩容、分库分表到异步降级，准备一套标准答题框架。

### Day 11（5.30）：全真模拟面试（一）
- 上午：回顾前 10 天的 Obsidian 错题库。
- 下午：45 分钟深度模拟面。不仅问技术，还要问“有没有观点冲突经历”“哪个项目收获最大”等软素质问题。
- 晚上：复盘录音，修复话术漏洞。

### Day 12（5.31）：全真模拟面试（二）
- 上午：针对昨天的弱点进行定点修复。
- 下午：45 分钟压力面试。不断打断、质疑技术选型，测试抗压能力和高情商兜底话术。
- 晚上：心态建设，梳理最终版本的自我介绍和“有什么想问我的”问题准备。

### Day 13（6.01）：静心与总复习
- 任务：轻复盘、轻回顾、稳心态，不再引入新的大块知识点。
- 核心输出：最终版自我介绍、项目话术总表、考前速记表。

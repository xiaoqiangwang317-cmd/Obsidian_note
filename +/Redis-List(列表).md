## 1. 介绍
List列表是简单的字符串列表，**按照插入顺序排序**，从头或者尾部添加。
## 2. 内部实现
List类型的底层数据结构是[[双向链表]]或[[压缩列表]]实现
但是在 Redis 3.2 版本之后，List 数据类型底层数据结构就只由[[ quicklist ]]实现了，替代了双向链表和压缩列表；而从 Redis 7.0 起，quicklist 节点内部用来保存元素的压缩列表也被 [[listpack ]]替代。
## 3. 应用场景

> [!NOTE] 消息队列
> 消息队列在存取消息时，必须要满足三个需求，分别是**消息保序、处理重复的消息和保证消息可靠性**
> 1. 满足有序
> 本来LIst就有序，可以生产者Lpush，消费者RPOP。
> 为了不让消费者一直等待获取，Rdeis使用[[BRPOP]]指令
> 2. 处理重复消息
> - 每个消息要有一个全局ID
> - 消费者要记录处理过的消息ID，遇见重复不在处理
>**List 并不会为每个消息生成 ID 号，所以我们需要自行为每个消息生成一个全局唯一ID**
>3.  消息可靠
>当消费者宕机时，消息被删除了但是没有被处理，为了留存消息提供[[BRPOPLPUSH]]指令
>
>缺点
>- 不支持多个消费者消费同一条消息
>- 不支持[[消费组]]实现


> [! Node]
> LPUSH key value      # 左侧插入
> RPUSH key value      # 右侧插入
> LPOP key             # 左侧弹出
> RPOP key             # 右侧弹出
> LRANGE key 0 -1      # 查看全部

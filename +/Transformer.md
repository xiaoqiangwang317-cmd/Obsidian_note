### 1. Self-Attention 机制的本质：信息路由与聚合

在底层，Self-Attention 是一种**基于特征相似度的数据路由机制**。你可以用熟悉的“数据库查询”逻辑来理解它：

- **$Q$ (Query - 查询向量)**：当前 Token 想要寻找什么特征。
    
- **$K$ (Key - 键向量)**：其他 Token 具备什么特征（相当于数据库索引）。
    
- **$V$ (Value - 值向量)**：其他 Token 携带的实际内容载体。
    

**计算逻辑：** $\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$

- **为什么要除以 $\sqrt{d_k}$（缩放因子）？**
    
    这是一个纯粹的数学推导问题。假设 $Q$ 和 $K$ 中的元素是独立的随机变量，均值为 $0$，方差为 $1$。
    
    当计算它们的点积 $q \cdot k = \sum_{i=1}^{d_k} q_i k_i$ 时，结果的均值依然是 $0$，但方差会累加扩大到 $d_k$。
    
    当维度 $d_k$ 很大（例如 128）时，点积的方差极大，数值会分布在极其宽泛的区间。将其输入 Softmax 函数后，最大值对应的概率会无限逼近 $1$，其余逼近 $0$。这会导致[[梯度消失（Gradient Vanishing）]]，模型无法有效更新参数。除以标准差 $\sqrt{d_k}$，是为了将点积结果的方差强制拉回 $1$，保持梯度的健康流动。
    
- **时间复杂度的灾难：**
    
    计算 $QK^T$ 时，是对维度为 $N \times d$ 的矩阵和 $d \times N$ 的矩阵进行乘法，生成一个 $N \times N$ 的注意力分数矩阵（Attention Map）。这步乘法的时间复杂度是 $O(N^2 \cdot d)$。随着上下文长度 $N$ 的增加，计算量呈**平方级爆炸**，这是目前长文本大模型面临的最大算力瓶颈。
    

### 2. 位置编码 (Positional Encoding)：打破置换不变性

纯粹的 Attention 计算本质上是**集合操作**（类似于词袋模型），无论输入 Token 的顺序怎么换，只要特征相同，点积算出来的分数是一样的。但这在自然语言中是致命的（“猫吃老鼠”和“老鼠吃猫”语义完全相反）。

- **绝对位置编码 (Sinusoidal)：**
    
    早期的做法，通过正弦和余弦函数，给每个绝对位置（1, 2, 3...）生成一个固定的、不依赖数据的向量，直接加到输入 Token 的 Embedding 上。
    
- **旋转位置编码 (RoPE - Rotary Position Embedding)：**
    
    这是目前主流的解法（LLaMA, Qwen 均采用）。RoPE 的高明之处在于，它用**绝对位置的变换，实现了相对位置的表达**。
    
    在计算点积之前，通过一个特定的旋转矩阵对 $Q$ 和 $K$ 进行旋转。数学上可以证明，旋转后的两个向量的点积，仅仅取决于它们在序列中的**相对距离** $(m-n)$。相对距离越远，注意力衰减越明显。这种机制使得模型对长文本的泛化和外推能力大幅增强。
    

### 3. KV Cache：空间换时间的极限推演

这是纯粹的后端并发优化思维在大模型推理上的应用。

大模型的推理（生成过程）是**自回归 (Autoregressive)** 的，即：生成第 $t$ 个词时，需要看前面 $t-1$ 个词；生成第 $t+1$ 个词时，需要看前面 $t$ 个词。

- **如果没有 KV Cache：**
    
    为了生成第 $t$ 个 Token，系统需要把前 $t-1$ 个 Token 重新过一遍完整的 Self-Attention 计算。大量的时间被浪费在重复计算历史信息的 $K$ 和 $V$ 上，使得单步推理的时间复杂度高达 $O(N^2)$。
    
- **KV Cache 的工程解法：**
    
    既然历史 Token 的特征是固定的，那就在计算完之后，把它们的 $K$ 和 $V$ 矩阵缓存在显存中。
    
    当生成第 $t$ 个 Token 时，只需要计算当前 Token 自己的 $q_t, k_t, v_t$。
    
    然后，用当前的 $q_t$（维度为 $1 \times d$）去点乘**缓存中**的 $K_{1:t-1}$ 以及刚刚生成的 $k_t$。
    
    这直接将单次生成的计算时间复杂度从 $O(N^2)$ 降维到了 $O(N)$。
    

代价是什么？**显存爆炸**。随着并发用户数增多和上下文变长，显存会被海量的 KV Cache 占满，导致系统出现 OOM（Out Of Memory）。所以目前业界才会有 PagedAttention（类似操作系统的虚拟内存分页）等底层显存调度优化技术。

 项目经历 

[[基于大模型的代码智能体系统 (AI Coding Agent) 独立开发]] 
项目简介：设计并实现一套以大模型为核心的代码智能体系统，使模型具备代码理解、 任务执行与工具调用能力。系统通过任务调度与执行隔离机制支持复杂开发任务的处 理，可自动化完成代码编写、文件操作、项目构建等开发任务。 
技术栈：Python、LLM (Qwen)、自研 Agent 架构、Tool Calling、MCP 协议、Git Worktree 核心工作： 
• 设计并实现 Agent 执行闭环 (Agent Loop)：实现推理-工具调用-结果校验的全流程 自动化，支持将复杂开发任务拆解为 3-5 步执行链，提升任务自动化能力。 • 封装文件操作与命令执行等 Tool Calling 能力：实现 Task 调度与状态管理，支持 多步骤任务追踪、异常重试与执行日志记录。 
• 基于 Git Worktree 实现多任务并行执行与执行环境隔离：支持 2-4 个任务并发运 行，有效避免分支冲突。 
• 基于 MCP+Tool Router 构建插件化扩展体系：统一接入外部工具并配置权限校验， 提升系统扩展性与安全性。

[[基于多 Agent 协同的智能客服系统]] 
 项目简介：基于大语言模型构建智能客服系统，设计多 Agent 协同框架，实现用户问 题理解、任务拆解、知识检索与答案生成的闭环流程。系统结合 RAG 机制，提升复 杂业务场景下的问答准确性与稳定性，支持多轮对话与上下文记忆。 
 技术栈：Python、FastAPI、RAG、FAISS、Prompt Engineering、MySQL 核心工作：
  • 基于手写状态机设计多 Agent 协同工作流：摒弃传统 LangChain 黑盒框架，采用原 生 Python 底层构建调度引擎，拆分意图理解、知识检索、答案生成等模块，实现 任务调度与状态精细化管理，提升系统扩展性与复杂任务处理能力。
  • 搭建 Embedding+FAISS 向量检索系统：完成业务知识库向量化存储与高效检索， 结合大模型生成，显著降低回答幻觉，提升问答准确率。 
  • 针对不同 Agent 设计差异化 Prompt 策略：优化多轮对话理解能力，并通过 Prompt 调优降低模型幻觉问题。
  • 使用 FastAPI 搭建标准化服务接口：基于 MySQL+Redis 实现用户会话管理、上下 文记忆与状态持久化，支持高可用对话服务。

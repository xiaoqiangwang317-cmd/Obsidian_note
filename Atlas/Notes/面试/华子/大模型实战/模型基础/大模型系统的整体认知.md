# DeepSeek版

### 一、大模型的“金字塔”式系统架构（从下到上） 
#### 第 0 层：算力底座（基础设施层） 
* **硬件资源**：高性能 GPU 集群（如 H100, A100, 国产昇腾/沐曦等）、高速互联（NVLink, InfiniBand）、分布式存储（存模型权重和 Checkpoint）。 
* **并行计算策略**：数据并行（DP）、张量并行（TP）、流水线并行（PP）、序列并行（SP）。
* **核心难点**：**显存墙**和**通信墙**。 
* **关键技**：**ZeRO（分片优化器/梯度/参数）**，FlashAttention（加速注意力计算）。 
#### 第 1 层：数据工程层（大模型的“矿藏”） 
* **数据采集**：互联网爬虫、公开数据集、私有数据。 
* **数据处理**：去重、去毒、去 PII（个人隐私信息）、质量过滤、格式清洗。 
* **数据构建**：**预训练数据**（万亿 Tokens 的文本/代码/多模态对） + **指令微调数据**（SFT 数据：问题-答案对） + **强化学习偏好数据**（RLHF 数据：人类对回答的排序）。 
* **核心法则**：**Scaling Law（尺度定律）** —— 数据质量与规模的胜利。
#### 第 2 层：模型算法层（大脑的“结构”） 
* **架构选择**：纯文本 LLM：Transformer 架构（自回归）。 
* 多模态 LLM：CLIP（视觉编码器）+ LLM 组合（如 LLaVA），或原生多模态（如 Gemini）。 
* **核心设计**： 
	* Attention 机制
	* MoE (Mix of Experts) 
	* Positional Embeddings (RoPE, ALiBi) 
	* Tokenization (分词器)
* **训练范式**：**自监督学习**（Next Token Prediction）。 
 #### 第 3 层：训练工程层（把矿炼成金） 分为三大关键阶段： 
1. **预训练 (Pre-training)**：从零训练基础模型。 * 耗时最长、算力最大、最昂贵。 * 解决通用“理解与生成”能力。 * 问题：**显存爆炸** → 使用 **ZeRO-3**，模型并行。 
2. **微调 (Fine-tuning)**：把通用模型变成业务专家。 * 指令微调（SFT）：让模型听得懂人话（“你是谁？”→“我是AI助手”）。 * 人类反馈强化学习（RLHF）：让模型变“乖”、无害、对齐人类价值观。 * 参数高效微调（PEFT）：如 **LoRA**（只更新少量参数，省显存）。 
3. **后训练/对齐**：PPO, DPO（直接偏好优化）等。 
#### 第 4 层：模型优化与部署层（把它推到用户面前） 
* **模型压缩**： 
	* **量化 (Quantization)**：把模型从 FP16 压缩到 INT4/INT8，显存骤降，速度飙升（如 AWQ, GPTQ）。 
	* **剪枝 (Pruning)**：去掉不重要的神经元。 
	* **蒸馏 (Distillation)**：大模型教小模型。 
* **推理加速**： 
	* **KV Cache**：存储过去生成的 Key/Value，避免重复计算。 
	* **Speculative Decoding (投机采样)**：用小模型兜底，大模型验证，提速。 
	* **vLLM / TGI / TensorRT-LLM**：专门的高性能推理框架。 
* **部署形态**： 
	* 云端 API（如 OpenAI, DeepSeek）。 
	* 本地私有化部署（如 Ollama, vLLM 本地服务）。 
	* 边缘端推理（手机/PC）。 
#### 第 5 层：应用与交互层（最终的价值） 
* **Prompt Engineering（提示词工程）**：最轻量级的“调教”。 
* **RAG (检索增强生成)**：把外部知识库（PDF, 数据库）当“参考书”喂给模型。 
* **Agent（智能体）**：让大模型能**调用工具**（搜索、计算器、Python 代码、数据库）。 
* **Workflow（工作流）**：拆解复杂任务，交给模型协作完成。 
#### 第 6 层：评测与安全（刹车） 
* **评测基准**：MMLU（通用知识），GSM8K（数学），HumanEval（代码）。 
* **安全性**：越狱防御（Jailbreak Defense），有害内容过滤，红队测试。

---

## 1. Token 是什么

比如：

```
你好世界
```

不是直接输入模型。

而是：

```
15120 3837 9910 ...
```

模型永远只认识：

# token id

---

## 2. Embedding 是什么

token id：

```
15120
```

会变成：

一个高维向量：

x∈Rdx\in\mathbb{R}^dx∈Rd

这一步：

叫：

# embedding

本质：

> “把文字映射到向量空间”。

---

## 3. Attention 是什么（最重要）

这是 Transformer 核心。

一定要理解：

为什么：

```
“它”
```

能知道指代谁。

核心：
$$
\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

你现在不用硬背。

但一定要知道：

它本质是在：

# 计算 token 与 token 的关系强度。

---

## 4. 为什么会有 KV Cache

这个直接关系：

- 推理速度
- 显存
- 长上下文

这个是：

大模型工程里极其核心的概念。

---

## 5. 下一 Token 预测

模型本质不是“思考”。

而是：

# 预测下一个 token 概率。

例如：

```
中国的首都是
```

模型会算：

```
北京：0.92 上海：0.03 广州：0.01
```
